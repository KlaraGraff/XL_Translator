# 翻译记忆库维护说明

最后更新：2026-07-24

## 当前基线

新版在 `~/Library/Application Support/Translator/tm.db` 使用当前翻译记忆库格式。首次使用时会创建空数据库与当前表结构；不会探测、读取、备份、迁移、修复或删除旧版本的 TM 数据库。

如果当前新基线数据库的 schema 损坏、属于未来版本或无法安全识别，应用不会静默修复或覆盖它。维护页会将该类别标为不可写，用户可以先主动导出当前可读内容，再明确确认清空该类别或完整本地重置。

## 正常维护

1. 在应用内“维护”页查看 TM 占用、条目数与 schema 状态。
2. 使用模块 2 的当前格式 JSON 导出作为主动备份；导出包含完整术语内容，应按敏感文档保管。
3. 需要清理时，只能按语言对或全部清空，并在条目数与二次确认后执行。
4. TM 清理、清空或完整本地重置都不会删除源文件、用户输出目录或已生成翻译文件。

## 不要使用的旧路径

以下旧迁移或修复假设不属于当前版本，不应执行：

- `core/data_migration.py`
- 自动迁移 `~/.xl_translator` 中的旧 TM、设置或 Key
- `python src/scripts/repair_db.py`
- `streamlit run app.py`
