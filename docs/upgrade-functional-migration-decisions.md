## 2. 迁移原则

1. 先逐模块完成决策对齐，再制定实施方案；未经用户确认不得修改业务代码。
2. 本轮不兼容本次新基线之前的设置、模型配置导出、TM 数据或自定义语言数据；本次新版正式发布后形成新的数据基线，后续版本必须为该基线提供向前升级。
3. 本轮优先保障 macOS 用户体验；新版最低支持 macOS 12.0 Monterey，并分别提供 Apple Silicon `arm64` 与 Intel `x64` 原生安装包。Windows 用户暂继续使用旧版本，GitHub 自动打包只保留 macOS。
4. 不因 Tauri 迁移而缩减已经确认仍有使用价值的功能。
5. 对自动化结论宁可拒绝复用并正常翻译，也不得以猜测语言或错误 TM 命中换取速度。
6. 每个已启动任务必须冻结自身的语言、模型、API Key、吞吐和输出设置快照；后续编辑只影响未来任务。

### 6.3 模块 2A 决策摘要

- 自动翻译只写真实正向语言对；不自动生成反向词条。
- 用户手动新增或编辑内置语言对时可选择同步反向词条，默认不勾选；发生反向冲突时不覆盖；自定义目标语言不提供反向同步。
- 自动模式入库遵循文件预检、逐条语言回报和质量门；手动模式以用户源语言为权威并校验模型回报；不确定、混合或不一致内容不污染 TM。
- TM 采用五级可信状态；普通自动译文出现不同结果时进入冲突候选，不再静默覆盖。
- 清洗永远先出建议；用户确认后升级为已审核自动或清洗锁定。
- 本轮不兼容旧 TM 数据；旧库的双向回填和自动同步逻辑将在实施中按本决策替换。
